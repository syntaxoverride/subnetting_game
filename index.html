Subnetting Help

What is Subnetting?

Subnetting is the process of dividing a large network into smaller, more manageable pieces called subnets. Think of it like dividing a large building into separate floors — each floor (subnet) has its own range of room numbers (IP addresses).

Key Concepts

  • IP Address: A unique 32-bit number (e.g., 192.168.1.1) that identifies a device on a network.
  • Subnet Mask: Tells you which part of the IP is the "network" portion and which part is for "hosts" (devices).
  • CIDR Notation: A shorthand for the subnet mask. /24 means 24 bits for the network, 8 bits for hosts.
  • Network Address: The first address in a subnet — identifies the subnet itself (host bits all 0).
  • Broadcast Address: The last address in a subnet — used to send to all hosts (host bits all 1).
  • Usable Hosts: Total addresses minus 2 (network and broadcast are reserved).

How to Subnet — Step by Step

  1. Determine how many hosts you need — This is given in each question.
  2. Calculate host bits needed — Find the smallest n where 2n ≥ hosts + 2.
  3. Calculate the CIDR prefix — prefix = 32 - n.
  4. Find the subnet mask — Convert the prefix to dotted decimal.
  5. Find the network address — Apply the mask to the given IP (host bits become 0).
  6. Find the first usable IP — Network address + 1.
  7. Find the broadcast address — Network address with host bits all set to 1.
  8. Find the last usable IP — Broadcast address - 1.

Worked Example

Problem: You need 50 hosts. Your address block is 192.168.1.0.

Step 1: 50 hosts needed.

Step 2: 26 = 64 ≥ 50 + 2 = 52. So n = 6 host bits.

Step 3: CIDR prefix = 32 - 6 = /26.

Step 4: Subnet mask = 255.255.255.192.

Step 5: Network address = 192.168.1.0 (already aligned).

Step 6: First usable = 192.168.1.1.

Step 7: Broadcast = 192.168.1.0 + 63 = 192.168.1.63.

Step 8: Last usable = 192.168.1.62.

Tips for Success

  • Memorize the common powers of 2: 2, 4, 8, 16, 32, 64, 128, 256.
  • The "block size" is 2(32 - prefix). Network addresses always fall on multiples of the block size.
  • Network address + block size - 1 = broadcast address.
  • Fill in all 4 fields (including Network and Broadcast) for maximum points!
